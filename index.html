TV Overlord is a semi automatic command line tool to download and manage TV shows from newsgroups or bittorent. It will download nzb files or magnet links.

TV Overlord is an open source application written in python3 and it's installed on the command line:

pip3 install tvoverlord — More details here.

Latest version: — Changes

 

It can show episodes ready to download.

It searches multiple sites simultaneously and combines the results into one list to select from.

TV Overlord keeps track of which shows have been downloaded and what shows are available to download.

For each new episode of a tv show you are tracking, you are given a list of possible downloads to choose. If you use a torrent search provider, a magnet link is passed to the default bittorent client. If it is an NZB search, an NZB file is placed in a folder that is configured in the configuration file.

For torrent files, you can also have your shows organized after downloading. If you use transmission or deluge, those clients can be configured to call a script when each torrent is complete. This script will extract the video file from the downloaded folder, rename it, and put it in a separate folder organized into sub folders named for each tv show. Get more information about the Rename tags here.

There are several bittorent search providers and two NZB search providers and new ones can be added fairly easily. See the search providers README.org, or by making a feature request in issues or a pull request.

Overview

  • Keeps track of downloaded shows.
  • Show a list of shows available for download.
  • Gives you a list of torrents or nzbs you can choose from.
  • You can add new shows from the command line.
  • Warn you when not using a vpn.
  • Will display info showing what show will be next and how many days till broadcast. This list can be filtered and sorted in various ways.
  • Displays a calendar of upcoming episodes. You can also specify a range of days to display, past or future.
  • The ability to query download history and re-start a download.
  • Organize you tv show downloads into folders.

Get the detailed command line reference here.

Why semi automatic

Semi automatic means that you manually run it and select from a list each episode you want. The reason for this is that not all torrents follow a naming convention that is parsable, especially for more obscure shows. So instead of writing a ton of code to try and figure out information about an episode, I’ll use the parser built into my brain. Also I like that it runs when I like, its light weight, and there are no background processes.

Changes

Version 1.4.7, 2016-12-14

  • Tvol now supports shows that use dates in their torrent name instead of season and episode. Docs here.
  • 1337X rewrote their html and the scraper works again.
  • Added more Bit Torrent Scene url's.
  • Emergency fix for ExtraTorrents malformed feed.

Version 1.4.5, 2016-11-11

  • Change download progressbar to "status lights".

    The progressbar was the wrong tool since it implied that searching was happening sequentially when it is actually searching in parallel. This also shows which search engines are the slowest.

  • Added a search engine exclude option to downloading.

    Using tvol download --exclude pb the piratebay would be excluded from searches.

    This can be a comma separated list as well and if there are spaces they should be wrapped in quotes. The long and short names can be used. You can find these with the config command under "Search engines". Some examples:

    • pb
    • PB,rb,13
    • 'the pirate bay, rarbg, ez'

Version 1.4.4

  • Adding new shows has been streamlined.
  • All error messages are going to stderr instead of everything going to stdout.
  • getipintel.com is no longer being used to find out if the user is connected to a VPN. Now, an IP whitelist is being stored in the db. More information here.

Version 1.4.3

  • Users can opt-in to supply telemetry data
  • tvol will notify the user if a newer version is available.
  • Multiple configs can be used by using named configs.
  • qBittorent is now supported

Version 1.4.1

  • tvol nondbshow SEARCHSTRING with no matches, no longer causes error.
  • Remove non relevent options from nondbshow.
  • Remove '[enter] for #1' from all interactive ui's.

Version 1.4.0

  • When editing a show with tvol editshow <show name> and more that one match is found, a list is shown to choose from instead of using the first match.
  • After adding a show with tvol add <show name> you are asked which episode to start downloading at; first, last or a season and episode.
  • When adding a new show and more than one match is found, you are given a list to choose from instead of using the first match.
  • An optional blacklist field has been added to config.ini. Any of the search engines you don't want to use, they can be added to a comma separated list. More information here
  • Bulk adding shows is possible now with the --bulk flag, tvol add --bulk <show name> --season 2 --episode 3. If season or episode are not used, they default to 0. Also when using --bulk, no checks are done. More information here
  • A bug that prevented using shows with more than 99 episodes has been fixed.
